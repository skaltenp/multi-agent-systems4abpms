5. **Enriched data: join with EABLG, net_master, sales_master**

Created `J09A_enriched` as the export-ready table:

- `EABLG` join via installation:
  - `J09A_temp.ANLAGE` = `eablg.Installation`
  - Pulls `"Meter Reading (MR) Doc. No."`.

- `net_master` and `sales_master` join via `EQUNR`.
  - For each field, `COALESCE(sales, net)` is used, so:
    - If **both** net and sales exist for a given `EQUNR`, the **sales** data is used and net is effectively ignored.
    - If **only net** exists, the net data is used.
    - This matches your preference rule.

```sql
DROP TABLE IF EXISTS J09A_enriched;

CREATE TABLE J09A_enriched AS
WITH base AS (
    SELECT t.*, e."Meter Reading (MR) Doc. No."
    FROM J09A_temp t
    LEFT JOIN eablg e
        ON t.ANLAGE = e.Installation
),
net AS (
    SELECT
        EQUNR,
        TITLE AS n_TITLE,
        FORENAME AS n_FORENAME,
        SURNAME AS n_SURNAME,
        STREET AS n_STREET,
        STREETNO AS n_STREETNO,
        POST_CODE1 AS n_POST_CODE1,
        CITY1 AS n_CITY1,
        CITY2 AS n_CITY2,
        EMAIL AS n_EMAIL
    FROM net_master
),
sales AS (
    SELECT
        EQUNR,
        TITLE AS s_TITLE,
        FORENAME AS s_FORENAME,
        SURNAME AS s_SURNAME,
        STREET AS s_STREET,
        STREETNO AS s_STREETNO,
        POST_CODE1 AS s_POST_CODE1,
        CITY1 AS s_CITY1,
        CITY2 AS s_CITY2,
        EMAIL AS s_EMAIL
    FROM sales_master
)
SELECT
    b."Meter Reading (MR) Doc. No.",
    b.HAUS,
    b.ANLAGE,
    b.ME_MA_ID,
    b.EQUNR,
    b.HOUSE_NUM1,
    b.ENTSCHEIDUNG,
    COALESCE(s.s_TITLE,   n.n_TITLE)    AS TITLE,
    COALESCE(s.s_FORENAME,n.n_FORENAME) AS FORENAME,
    COALESCE(s.s_SURNAME, n.n_SURNAME)  AS SURNAME,
    COALESCE(s.s_STREET,  n.n_STREET)   AS STREET,
    COALESCE(s.s_STREETNO,n.n_STREETNO) AS STREETNO,
    COALESCE(s.s_POST_CODE1,n.n_POST_CODE1) AS POST_CODE1,
    COALESCE(s.s_CITY1,   n.n_CITY1)    AS CITY1,
    COALESCE(s.s_CITY2,   n.n_CITY2)    AS CITY2,
    COALESCE(s.s_EMAIL,   n.n_EMAIL)    AS EMAIL
FROM base b
LEFT JOIN net   n ON b.EQUNR = n.EQUNR
LEFT JOIN sales s ON b.EQUNR = s.EQUNR;
```

Note: Due to the join with `eablg`, a single `J09A_temp` row can appear multiple times in `J09A_enriched` if there are multiple MR documents for the same installation; this is reflected in the export row counts below.
