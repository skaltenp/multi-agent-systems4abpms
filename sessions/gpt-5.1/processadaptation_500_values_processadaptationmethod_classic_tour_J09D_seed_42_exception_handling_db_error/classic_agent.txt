## 2. Add ENTSCHEIDUNG and classify each house

Logic:

- Default:
  - `< 3` EQUNR → `KSA`
  - `>= 3` EQUNR → `EVU`
- Overrides (take precedence over counts):
  - `HOUSE_NUM1 = 0` → `EVU`
  - `HOUSE_NUM1 BETWEEN 500 AND 599` → `EVU`

```sql

-- 3) Classify each row in J09D_temp
UPDATE J09D_temp
SET ENTSCHEIDUNG = CASE
    WHEN HOUSE_NUM1 = 0 THEN 'EVU'
    WHEN HOUSE_NUM1 BETWEEN 500 AND 599 THEN 'EVU'
    WHEN (
        SELECT COUNT(DISTINCT t2.EQUNR)
        FROM J09D_temp t2
        WHERE t2.HAUS = J09D_temp.HAUS
    ) < 3 THEN 'KSA'
    ELSE 'EVU'
END;
```

This version uses a correlated subquery so it works on standard SQLite.

## 3. Enrich data with EABLG, net_master, and sales_master

Assumptions (adjust column names if your schema differs):

- `J09D_temp` has `ANLAGE` (installation ID).
- `EABLG` has `INSTALLATION` and is joined by `EABLG.INSTALLATION = J09D_temp.ANLAGE`.
- `net_master` and `sales_master` both contain contact fields  
  `TITLE, FORENAME, SURNAME, STREET, STREETNO, POST_CODE1, CITY1, CITY2, EMAIL`
  and are joined by `EQUNR`.

Preference rule:

- If a `sales_master` row exists for an `EQUNR`, use its contact fields.
- Only if there is no `sales_master` row, fall back to `net_master`.

```sql
-- 4) Build the final result set with preferred customer contact data
DROP TABLE IF EXISTS J09D_final;

CREATE TABLE J09D_final AS
SELECT
    d."Meter Reading (MR) Doc. No." AS "Meter Reading (MR) Doc. No.",
    d.HAUS,
    d.ANLAGE,
    d.ME_MA_ID,
    d.EQUNR,
    d.HOUSE_NUM1,

    -- Prefer sales_master over net_master
    COALESCE(s.TITLE,      n.TITLE)      AS TITLE,
    COALESCE(s.FORENAME,   n.FORENAME)   AS FORENAME,
    COALESCE(s.SURNAME,    n.SURNAME)    AS SURNAME,
    COALESCE(s.STREET,     n.STREET)     AS STREET,
    COALESCE(s.STREETNO,   n.STREETNO)   AS STREETNO,
    COALESCE(s.POST_CODE1, n.POST_CODE1) AS POST_CODE1,

    COALESCE(s.CITY2,      n.CITY2)      AS CITY2,
    COALESCE(s.EMAIL,      n.EMAIL)      AS EMAIL

FROM J09D_temp d
```
